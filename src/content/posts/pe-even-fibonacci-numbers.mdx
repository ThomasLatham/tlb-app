# Introduction

This Project Euler problem, the second in the series, is rated at just 5% difficulty, but it still
took me quite a bit of time (and rhythmic, forceful meetings between my head and the wall) to come
up with and prove a solution with which I was satisfied. After taking the route which to me seemed
the most straight forward, I had a strong feeling that finer-tuned solutions were out there, so I
set out to see if I could find one.

The first stop in my quest for a more elegant algorithm was the exploration of a neat pattern I
noticed in the Fibonacci sequence, whose existence I ended up proving by induction after some trial
and error. I thought this would give me what I was after, but I soon realized that even when
applying this now-proven pattern, the computation for the problem's answer still took exactly as
many steps as my first method. I was about ready to give up there, feeling that my hard work on the
proof I had just finished was mostly for naught, but then I noticed a separate pattern in the
sequence of _even_ Fibonacci numbers. With this pattern I knew, if I could prove it did indeed hold
for the whole sequence, that it would allow me to implement a faster algorithm, even if just by a
constant factor.

With renewed enthusiasm, I set up a way to prove the new pattern and eventually got it all worked
out. The cool thing about that though (at least I thought so), was that I ended up using — and maybe
even needing — the proof of the first pattern in order to prove the second one. It just goes to show
that even when your initial plans don't land you exactly where you were expecting, sometimes simply
exploring something, whether it's a pattern, or a new hobby or even a fork in some trail you haven't
yet taken, sometimes that can turn out to be exactly what you needed after all.

# Problem Statement

**Even Fibonacci Numbers:**

_Each new term in the Fibonacci sequence is generated by adding the previous two terms. By starting
with $\textit 1$ and $\textit 2$, the first terms will be:_

# My Intuitive Approach

My initial approach to this problem was to iteratively generate the Fibonacci numbers, checking in
each step whether the current number is even, and if so adding it a running sum. You can see the
source code for my implementation of this solution
[here](https://github.com/ThomasLatham/project-euler/blob/main/src/evenFibonacciNumbers/evenFibonacciNumbers.ts)
and in the code snippet below:

```TypeScript
/**
 * Sums the even Fibonacci numbers, up to and including the given `maxAddend`.
 *
 * @param maxAddend The maximum addend to be added to the sum.
 * @returns The sum of all even Fiboannci numbers less than or equal to the `maxAddend`.
 */
const sumEvenFibonacciNumbers = (maxAddend: number): number => {
  let prev: number = 1;
  let cur: number = 1;
  let sum: number = 0;

  while (cur <= maxAddend) {
    sum += cur % 2 == 0 ? cur : 0;
    const temp = cur;
    cur += prev;
    prev = temp;
  }

  return sum;
};
```
